REST API

We have our "CRUD" operations in the "RestService". The "RestService" has access to the "Facade" class, which has a connection to the database. The RESTful API is just a simple collection of URIs, HTTP calls to those URIs and some JSON representations of resoursces, many of which contain relational links. Each resouse has its own address or URI - every thing the server can provide is exposed as a resource.

Logged in Users only allowed to use User API and Logged in Admin only able to Use Admin API

METHOD & PATH DESCRIPTION

GET
api/admin/users Returns an Array of all users with role User.
api/currency/dailyrates/ Returns an JSON representation of the data we get from http://www.nationalbanken.dk/_vti_bin/DN/DataService.svc/CurrencyRatesXML?lang=en
api/currency/calculator/{amount}/{fromcurrency}/{tocurrency} Returns amount*(fromRate/toRate)

POST
api/createuser/createuser Creates a new user if provided with a valid json object. A person may be created with firstName and Password like this:
{
"userName" : "Discofever",
"password" : "123"
}

DELETE
api/admin/user/{name} Deletes a person with the given name Like this Url: */api/admin/user/{userName}

JSON Format

JSON (JavaScript Object Notation) is a data-interchange format. It is easy to read and write. It is easy for machines to parse and generate. It is based on a subset of the Javascript Programming Language. JSON is a collection of name/value pairs.
